遵循以下规则生成 Git 提交信息：

## 格式要求
<type>(<scope>): <简短描述>

## Type 类型
- feat: 新功能
- fix: 修复 bug
- refactor: 重构代码
- style: 样式或格式
- docs: 文档更新
- chore: 杂项任务
- test: 测试相关
- perf: 性能优化

## 描述规则
- 用中文描述，动词开头
- 不超过 50 个字
- 不以句号结尾

## 示例
feat(用户系统): 新增注册和登录接口
fix(商品详情): 修复图片加载失败问题
docs: 更新项目配置说明
