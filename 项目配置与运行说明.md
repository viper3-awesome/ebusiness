# 项目配置与运行说明

本指南详细说明了如何从零开始配置、运行并验证一个功能完备的 eBusiness 电商系统。该系统基于 **Spring Boot 3.2.4 + Vue 3 + Element Plus**，集成了 JWT 认证鉴权、Redis 缓存、商品浏览、购物车、订单管理、后台管理等功能。

---

## 一、技术栈与环境准备

### 技术架构

| 层级 | 技术 |
|------|------|
| 后端框架 | Spring Boot 3.2.4 + Spring Data JPA |
| 数据库 | MySQL 8.0 |
| 缓存 | Redis 6.0+ |
| 鉴权 | JWT (Hutool JWTUtil) + Redis Token |
| 前端框架 | Vue 3.4 + Vite 5 |
| UI 组件库 | Element Plus 2.7 |
| 图表 | ECharts 5.5 |
| Java 版本 | JDK 17+ |
| 构建工具 | Maven 3.6+ |

### 环境要求

在部署项目前，请确保已安装以下环境：

- **JDK**: 17 或更高版本
- **Maven**: 3.6.0 或更高版本
- **MySQL**: 8.0 或更高版本（需保持运行状态）
- **Redis**: 6.0 或更高版本（需保持运行状态，默认端口 6379，无密码）
- **Node.js**: 14.x 或更高版本

## 二、数据库初始化（关键步骤）

数据库的正确初始化是项目成功运行的基石。请严格按照以下顺序，在 Navicat 或任意 MySQL 客户端中执行操作。

### 1. 创建数据库

```sql
CREATE DATABASE ch16 CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 2. 导入表结构和初始数据

依次执行项目中的两个 SQL 文件：

**(1) 执行 `ebusiness-server/ch16.sql`**

该文件包含所有数据表的结构定义，包括：
- `ausertable` — 管理员用户表
- `busertable` — 前台注册用户表
- `goodstype` — 商品类型表
- `goodstable` — 商品信息表
- `carttable` — 购物车表
- `focustable` — 商品收藏表
- `orderbasetable` — 订单主表
- `orderdetail` — 订单详情表

默认已插入管理员账号：
- 用户名：`admin`
- 密码：`admin`

**(2) 执行 `ebusiness-server/init_data.sql`**

该文件包含示例数据：
- 6 个商品类型（手机数码、电脑办公、服装鞋帽、食品生鲜、图书音像、家用电器）
- 6 个示例商品

> **注意**：`ch16.sql` 中的表使用 UTF-8 编码（`utf8_unicode_ci`），请确保数据库字符集一致，避免中文乱码。

## 三、后端配置与启动

### 1. 修改数据库连接

打开 `ebusiness-server/src/main/resources/application.properties`，确认以下配置与你的本地 MySQL 环境匹配：

```properties
spring.application.name=ch16
server.servlet.context-path=/eBusiness
server.port=8443

# 数据库连接 (请修改为你的实际配置)
spring.datasource.url=jdbc:mysql://localhost:3306/ch16?useUnicode=true&characterEncoding=UTF-8&allowMultiQueries=true&serverTimezone=GMT%2B8
spring.datasource.username=root         # 修改为你的数据库用户名
spring.datasource.password=123456       # 修改为你的数据库密码
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver

# JPA 配置
spring.jpa.hibernate.ddl-auto=none
spring.jpa.show-sql=true
spring.jpa.properties.hibernate.format_sql=true
spring.jackson.serialization.indent-output=true

# 文件上传临时目录
spring.servlet.multipart.location=D:/data/apps/temp
```

### 2. Redis 配置

项目使用 Spring Boot 自动配置连接 Redis，默认连接 `localhost:6379`，无密码。如需自定义，在 `application.properties` 中添加：

```properties
spring.data.redis.host=localhost
spring.data.redis.port=6379
# spring.data.redis.password=  (如有密码请取消注释)
```

### 3. JWT 与 Redis 过期时间配置

`ebusiness-server/src/main/resources/conf/conf.properties`：

```properties
token.header=Authorization
token.secret=jwtsecurit
token.iss=com_chenheng
token.expiration=10       # Token 有效期 10 分钟
redis.expiration=1800     # Redis 登录状态有效期 1800 秒（30分钟）
```

### 4. 构建与启动

在 `ebusiness-server` 目录下打开终端：

```bash
# 清理并编译
mvn clean package -DskipTests

# 启动应用（需要在 ebusiness-server 目录下）
mvn spring-boot:run
```

或者直接在 IDE 中运行主类：
`ebusiness-server/src/main/java/com/ch/ch16(1)/Ch16Application.java`

看到控制台输出 Spring Boot 启动成功信息即表示后端启动完成。

**后端访问地址**：`http://localhost:8443/eBusiness`

### 5. 关键路径说明

> **重要**：源码目录名包含 `(1)` 后缀（如 `ch16(1)`、`common(1)`、`config(1)`），这是由于目录复制产生的。Maven 编译不受影响，但 IDE 编辑时请注意实际路径。

**项目主要结构**：

```
ebusiness-server/src/main/java/com/ch/ch16(1)/
├── Ch16Application.java              # 启动类
├── common(1)/
│   ├── config(1)/                    # 配置类（Redis、MyBatis、WebMvc、Interceptor）
│   └── sercurity/utils/              # 工具类（JWT、Redis、MD5）
├── entity/
│   ├── admin/                        # 管理端实体
│   └── before/                       # 前台实体
├── mapper/
│   ├── admin/                        # 管理端 Mapper
│   └── before/                       # 前台 Mapper
├── service/
│   ├── admin/                        # 管理端 Service
│   └── before/                       # 前台 Service
└── controller/
    ├── admin/                        # 管理端接口 (/api/admin/**)
    └── before/                       # 前台接口 (/api/before/**)
```



## 六、鉴权机制说明

### 6.1 鉴权流程

```
用户请求 → AuthInterceptor 拦截器
    ↓
检查是否有 @AuthIgrone 注解 → 是 → 直接放行（如登录、注册、获取验证码）
    ↓ 否
从请求头 Authorization 获取 Token
    ↓
验证 JWT 签名（JwtTokenUtil）
    ↓
检查 Redis 中 login_<username> 是否存在且未过期
    ↓
通过 → 续期 Redis 过期时间 → 放行
```

### 6.2 不需要鉴权的接口（@AuthIgrone）

| 接口 | 说明 |
|------|------|
| `/api/before/register` | 用户注册 |
| `/api/before/login` | 用户登录 |
| `/api/before/getcode` | 获取图形验证码 |
| `/api/admin/login` | 管理员登录 |
| `/api/admin/goods/getGoodsIndex` | 首页商品列表 |

### 6.3 Token 与 Redis 机制

- **JWT Token** 有效期 10 分钟（`token.expiration=10`）
- **Redis 登录状态** 有效期 30 分钟（`redis.expiration=1800`）
- 每次校验通过后 Redis 自动续期 30 分钟
- Token 存储方式：前端存入 `sessionStorage`
  - 前台用户 key：`bemail`
  - 管理员 key：`aname`

## 七、API 响应规范

所有接口返回值统一使用 `ResponseResult<T>` 包装，包含四个字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `result` | T | 业务数据 |
| `statusCode` | 枚举 | C200 / C401 / C404 / C405 / C500 |
| `resultType` | 枚举 | SUCCESS / MESSAGE / ERROR / NOT_FOUND |
| `msgId` | String | 消息标识（如 "A001" 用户不存在，"A002" 密码错误） |

## 八、常见问题排查

### 1. 数据库连接失败
- 检查 MySQL 服务是否启动
- 检查 `application.properties` 中的 `url`、`username`、`password` 是否正确
- 确认数据库 `ch16` 已创建且表已导入

### 2. Redis 连接失败
- 检查 Redis 服务是否启动（终端执行 `redis-cli ping` 应返回 `PONG`）
- 如有密码，需要在 `application.properties` 中配置 `spring.data.redis.password`

### 3. 前端请求跨域错误
- 确认后端已启动在 `http://localhost:8443`
- 检查 Vite 代理配置或后端 CORS 配置

### 4. 登录后仍然提示"未登录"
- Token 有效期仅 10 分钟，超时需重新登录
- Redis 中的登录状态过期（30 分钟无操作），需重新登录

### 5. 文件上传失败
- 检查 `spring.servlet.multipart.location=D:/data/apps/temp` 目录是否存在
- 如在 Linux/Mac 上部署，需要修改此路径并确保目录有写入权限

### 6. 前端控制台报 "Failed to resolve component"
- 项目中已将所有 `<strike>` 标签替换为 `<s>` 标签，若仍出现类似错误，检查模板中是否有其他过时的 HTML4 标签
